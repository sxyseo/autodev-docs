## 功能

- **草图编码代理**
    - 使用草图查看器的智能驱动编码工作流。
- **自动开发模式**
    - AutoSQL（需要数据库插件）。上下文感知的SQL生成。
    - AutoPage（React）。上下文感知的Web页面生成。
    - 自动测试。创建单元测试意图，自动运行单元测试并尝试修复测试。
    - 自动文档。自动生成文档。
- **Copilot模式**
    - AutoDev将帮助你查找bug，解释代码，跟踪异常，生成提交等。
    - 特定模式。基于你的代码上下文（如Controller, Service `import`），AutoDev将为你推荐最佳代码。
    - 相关代码。基于最近的文件更改，AutoDev将调用计算相似块来生成最佳代码。
- **与AI聊天**。与选择的代码和上下文感知的代码聊天。
- **自定义**。
    - 提示词的自定义规范。例如，Controller, Service, Repository, Model等。
    - 自定义意图操作。你可以添加自己的意图操作。
    - 自定义LLM服务器。你可以在`设置` -> `工具` -> `AutoDev`中自定义你的LLM服务器。
    - 自定义活文档。自定义你自己的活文档，如注释。
    - 团队AI。在代码库中自定义你的团队提示词，并分发给你的团队。
    - 提示词覆盖。你可以在代码库中覆盖AutoDev的提示词。
- **SDLC**
    - VCS。生成/改进提交消息，发布说明等。
    - 代码审查。生成代码审查内容。
    - 智能重构。基于AI的重命名，带有代码味道的重构，重构建议等。
    - Dockerfile。基于你的项目，生成Dockerfile。
    - CI/CD配置。基于构建工具，生成CI/CD配置文件，如`.github/workflows/build.yml`。
    - 终端。在终端工具窗口中，你可以使用自定义输入来生成shell/命令。
- **自定义AI代理**
    - 可执行AI代理语言：DevIns。
    - 自定义AI代理。你可以将自己的AI代理集成到AutoDev中。
- **模型**
    - 内置LLM微调
    - [UnitEval](https://github.com/unit-mesh/unit-eval) 评估llm结果
    - [UnitGen](https://github.com/unit-mesh/unit-gen) 生成代码-llm微调数据。
